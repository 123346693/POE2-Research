# 深度开发路线

## 阶段 1：可信推荐底座

- 建立职业/升华/玩法知识库。
- 建立 Build 候选数据契约，要求证据和版本号。
- 支持 Divine 预算约束，例如 `--budget-divines 5`。
- 支持角色快照诊断，例如 `diagnose --character ... --budget-divines 20`。

## 阶段 2：数据接入

- POE2DB/Wiki 快照导入器：技能、辅助、词缀、暗金、升华节点。
- 官方 API OAuth：用户角色导入，读取装备、技能、角色属性。
- trade2 查询计划：生成可复核的交易搜索；后续再接受控抓取和缓存。
- poe.ninja economy：唯一物品和经济趋势缓存。

## 阶段 3：Build 可行性验证

- 装备门槛：核心词缀、抗性、生命/ES、防御层、武器 DPS 或技能等级。
- 伤害门槛：按主技能类型估算基础伤害、more/less、暴击、异常、召唤物等。
- 生存门槛：抗性、护甲/闪避/格挡/回复/控制/距离，按玩法目标调权。
- 预算门槛：5D、20D、100D 分档，禁止推荐预算外“纸面神装”。

## 阶段 4：角色优化

- 输入人物链接或本地角色 JSON。
- 识别当前 Build 核心、装备短板、天赋方向、技能连接。
- 给出升级顺序：先修死亡原因，再补伤害，再做奢侈提升。
- 每条升级建议输出：预计花费、预计收益、交易搜索方案、置信度。

## 阶段 5：中文交互

- 用户直接说：“我想玩 Invoker，5D，刷图快，不要太脆。”
- 系统解析成结构化需求。
- 输出 Build、装备购买清单、技能链接、天赋方向、防御目标、后续升级路线。
